# 搜索结果卡片命中解释增强设计

## 背景

当前 SearchPage 的单条结果卡片已经提供：

1. 命中标签（双命中 / 关键词优先 / 语义辅助）
2. 来源标签（keyword / semantic）
3. 语义分数字段 chip
4. 排序依据
5. 语义命中摘要

这些信息是完整的，但用户仍然需要自己拼装判断：

- 这条结果到底为什么值得看？
- 这是强语义命中，还是辅助语义线索？
- 是关键词主导，还是关键词 + 语义共同支持？

## 目标

在每张搜索结果卡片上增加一层**轻量结论式解释**，让用户更快判断这条结果的命中质量。

## 方案

新增一个结果卡片级 helper，基于现有：

- `classifySearchResultHit`
- `classifySemanticExplanationTier`
- `semanticHitField`

生成单条 `resultCardExplanation` 文案。

## 文案规则

### 1. 双命中 + 高质量语义字段

输出：

- `这条结果同时命中关键词与高质量语义字段，可优先查看。`

### 2. 双命中 + 辅助语义字段

输出：

- `这条结果同时命中关键词，但语义信号主要来自辅助字段，建议结合预览确认。`

### 3. 关键词优先

输出：

- `这条结果主要由关键词命中进入结果。`

### 4. 纯语义 + 高质量语义字段

输出：

- `这条结果主要由高质量语义命中支持，适合优先检查。`

### 5. 纯语义 + 辅助语义字段

输出：

- `这条结果主要由辅助语义线索召回，建议谨慎判断。`

### 6. 其他情况

不显示额外结论文案。

## 展示位置

放在结果卡片中：

1. preview 文本之后
2. `排序依据` 之前

采用轻量容器样式，避免比“排序依据”更重。

## 结构原则

1. 解释生成逻辑放 helper 层
2. SearchPage 只负责渲染
3. 不引入新的复杂状态对象

## 非目标

本轮不做：

1. 多条解释并列
2. 颜色等级体系
3. 个性化推荐文案
4. 与 detail page 完整联动改造

## 测试设计

至少覆盖：

1. 双命中 + 高质量语义字段
2. 双命中 + 辅助语义字段
3. 关键词优先
4. 纯语义 + 高质量语义字段
5. 纯语义 + 辅助语义字段
6. SearchPage 结果卡片成功渲染该解释

## 完成后的转向建议

这一轮完成后，更适合转向：

1. detail page 命中上下文解释增强
2. 真正提升语义检索质量本身

不建议继续在结果卡片里无节制增加更多解释块。
