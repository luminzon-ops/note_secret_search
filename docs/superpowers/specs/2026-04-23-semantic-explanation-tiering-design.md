# 语义召回解释分层设计

## 背景

当前搜索页已经具备：

1. 关键词 / 语义 / 双命中的基础解释
2. 顶部检索链路总览
3. 结果项内的排序依据说明
4. 语义质量门槛与顶部质量门控说明

但当前解释仍然存在一个问题：

- 用户知道“有语义命中”，却不容易进一步区分这是更可信的强语义命中，还是只是辅助性的语义参与。

## 目标

为当前语义结果增加一层保守的解释分层，让用户更容易理解可信度层次：

1. 在顶部总览中表达高质量语义命中与辅助语义命中的数量
2. 在结果项解释中表达当前语义命中属于高质量还是辅助层

## 非目标

本次不做以下内容：

1. 不调整语义召回门槛
2. 不新增排序规则
3. 不做列表分区展示
4. 不改设置页
5. 不增加新的用户可调参数

## 方案选择

本次采用：

- **顶部总览 + 结果项同时体现**

但仍然坚持最小 MVP：只做解释分层，不增加新的底层筛选逻辑。

## 设计原则

### 1. 解释分层应复用现有信号

本轮不新增新模型信号，而是基于当前已存在的信息进行分层：

1. `semanticHitField`
2. `semanticScore`
3. 当前已有的高低优先级字段语义

### 2. 高质量语义命中与辅助语义命中都只是解释层

它们用于帮助用户理解结果可信度，不改变当前结果是否被召回。

### 3. 分层规则必须保守且易懂

优先级先按字段语义强弱来定，不做太复杂的阈值组合。

## 分层规则

### 高质量语义命中

满足以下任一条件即可视为高质量语义命中：

1. `semanticHitField` 属于高优先级字段：
   - `title`
   - `username`
   - `summary`

### 辅助语义命中

以下字段视为辅助语义命中：

1. `url`
2. `secretNote`
3. `tags`
4. `noteBody`

### keyword-only / dual-hit 关系

这轮不替换当前：

1. `双命中`
2. `关键词优先`
3. `语义辅助`

而是在语义相关解释里额外补一层可信度说明。

## 顶部总览

在 `SearchPage` 顶部检索链路总览中，当语义结果参与统一结果时，补充：

1. 高质量语义命中数量
2. 辅助语义命中数量

示例方向：

- `当前语义结果中，高质量语义命中 2 条，辅助语义命中 1 条。`

## 结果项解释

在结果项中：

1. 若该结果的语义命中属于高优先级字段，则显示“高质量语义命中”类表述
2. 若属于辅助字段，则显示“辅助语义命中”类表述

要求：

1. 复用现有 `排序依据` 区块
2. 保持文案简短
3. 不新增复杂视觉层级

## 结构建议

优先在：

- `lib/features/search/presentation/search_result_explanation.dart`

中集中新增分层 helper，避免把规则散进 `search_page.dart`。

## 测试设计

至少补齐以下测试：

1. 顶部总览能体现高质量 / 辅助语义分层数量
2. 结果项解释对高优先级语义字段给出“高质量语义命中”说明
3. 结果项解释对弱字段给出“辅助语义命中”说明
4. keyword-only 场景不误报语义分层说明

## 默认假设

1. 分层只影响解释层，不影响召回层。
2. 高优先级字段仍以 `title / username / summary` 为主。
3. 本轮重点是解释可感知化，而不是继续调质量门槛。

## 完成后的转向建议

这轮完成后，更适合转向：

1. 详情页承接语义分层解释
2. 语义召回可观测性（例如质量统计或结果来源分布）
