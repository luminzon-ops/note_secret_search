# Detail Hit Explanation Template Unification 设计

## 背景

现在详情页搜索 handoff card 已经具备三层信息：

1. `命中方式`
2. `命中说明`
3. `优先查看`

其中 `优先查看` 已经和列表页字段解释统一，但 `命中说明` 仍然直接回显原始上下文，例如：

- `标题：Bank Account`
- `摘要：恢复码备忘`
- `正文：backup codes`

这和列表页里已经更产品化的解释句式相比，仍然偏“原始数据回显”，不像完整解释链路的一部分。

## 目标

将详情页的 `命中说明` 从原始上下文回显，统一成与列表页 explainability 风格一致的句式模板。

## 方案

在 `detail_search_hit_target.dart` 中新增共享 helper，基于 `searchContext` 生成结构化命中说明：

- 标题 → `本次命中主要落在标题字段。`
- 账号 → `本次命中主要落在账号字段。`
- 网址 → `本次命中主要落在网址字段。`
- 标签 → `本次命中主要落在标签字段。`
- 备注/附注 → `本次命中主要落在备注字段。`
- 摘要 → `本次命中主要落在摘要字段。`
- 正文 → `本次命中主要落在正文内容。`

若无法识别字段，则继续回退到原始 `searchContext`，避免信息丢失。

## 结构原则

1. `命中说明` 与 `优先查看` 都来自 shared helper 层
2. detail page 不直接硬编码字段句式
3. 保留原始 `searchContext` 作为 fallback，不改变路由参数结构

## 非目标

本轮不做：

1. 改动列表页 `语义命中` 区块内容
2. 改动搜索结果路由参数
3. 增加更多解释维度
4. 改动高亮与滚动定位逻辑

## 测试设计

至少覆盖：

1. secret/note detail page 在已识别字段时显示新模板化 `命中说明`
2. `detail_search_hit_target.dart` 的 helper 能输出模板化说明
3. unknown context 继续回退显示原始值

## 完成后的转向建议

这一轮完成后，更适合转向：

1. 将 detail page `承接说明` 进一步细化到字段级保留原因
2. 统一列表页 `语义命中` 区块的句式模板
