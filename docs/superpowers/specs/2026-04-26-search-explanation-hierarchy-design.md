# Search Explanation Hierarchy Unification 设计

## 背景

上一轮已经统一了搜索链路里的核心术语，例如：

1. `语义命中`
2. `重点语义命中`
3. `补充语义线索`

但现在列表页和详情页对“为什么命中、先看哪里”的解释层级仍然分散：

- 列表页有 `排序依据`
- 详情页有 `命中说明`
- 详情页还有 `优先查看`

这些都在解释“本次命中是怎么来的”，但没有形成统一结构。

## 目标

把搜索列表和详情页的 explainability 进一步统一成同一层级：

1. 先说明命中来自哪里
2. 再说明这属于哪一类解释
3. 最后给出用户应该优先查看的位置

## 方案

本轮不改底层搜索逻辑，只统一 presentation helper：

1. 在 `detail_search_hit_target.dart` 中新增共享 explanation helper
2. 让详情页的 `命中说明` 与 `优先查看` 来自同一组规则
3. 让列表页 `排序依据` 尽量复用同一套字段级优先提示语言

## 统一后的解释层级

### 1. 列表页：排序依据

继续保留 `排序依据` 区块，但字段解释往“先看哪里”靠拢：

- 标题 → `优先查看标题，这是当前最直接的命中位置。`
- 账号 → `优先查看账号字段，这里最可能承载本次命中。`
- 网址 → `优先查看网址字段，确认站点是否与查询相关。`
- 标签 → `优先查看标签字段，确认标签线索是否匹配。`
- 备注/附注 → `优先查看备注字段，查看补充说明是否匹配查询意图。`
- 摘要 → `优先查看摘要，先确认概要是否对应本次查询。`
- 正文 → `优先查看正文内容，这里最可能包含本次命中上下文。`

### 2. 详情页：命中说明

保留原始 `searchContext` 展示，但增加 shared helper 产出的结构化提示，使详情页解释不再只是原样显示上下文。

### 3. 详情页：优先查看

详情页仍保留 `优先查看`，但文案完全来自共享 helper，与列表页字段优先提示一致。

## 结构原则

1. 共享规则放在 `detail_search_hit_target.dart`
2. 列表页与详情页使用同一字段解释 helper
3. SearchPage 不直接硬编码字段优先说明
4. Secret / Note detail page 不再各自维护一套 if-else 文案

## 非目标

本轮不做：

1. 改动搜索路由参数结构
2. 改动 semantic hit summary 生成逻辑
3. 新增更多视觉层级
4. 改动命中高亮逻辑

## 测试设计

至少覆盖：

1. 共享 helper 能把 secret/note 的 searchContext 解析为统一优先提示
2. SearchPage 的 `排序依据` 使用统一后的字段提示句式
3. SecretDetailPage / NoteDetailPage 的 `优先查看` 使用共享 helper 输出
4. 旧上下文别名（如 `备注：` / `附注：`）继续可用

## 完成后的转向建议

这一轮完成后，更适合转向：

1. 统一 `命中说明` 与 `语义命中` 区块的句式模板
2. 让 detail page 对 semantic-only retained 结果显示字段级保留原因
