# 搜索观测摘要分级展示设计

## 背景

当前 SearchPage 顶部的“搜索观测摘要”已经包含：

1. 命中结构
2. 语义分层
3. 字段分布
4. 主导信号提示
5. 主导字段提示
6. 提醒型结论

信息已足够有用，但默认全部展开会持续拉高 SearchPage 顶部高度。

## 目标

在不移除现有观测信息的前提下，让摘要默认更紧凑：

1. 默认只展示最关键的 2~3 行
2. 次级诊断信息放到“展开更多”后查看
3. 不引入复杂交互，不做 dashboard

## 默认展示策略

默认可见内容：

1. 命中结构
2. 主导信号提示
3. 提醒型结论（若有）

折叠内容：

1. 语义分层
2. 字段分布（若有）
3. 主导字段提示（若有）

## 交互

在“搜索观测摘要”块底部增加一条轻量按钮：

- 折叠态：`展开更多观测`
- 展开态：`收起观测详情`

点击后在当前卡片内展开 / 收起，不跳转、不弹窗。

## 结构原则

1. helper 层不承担 UI 折叠状态
2. `SearchObservabilitySummary` 继续只负责文本内容
3. 折叠状态留在 `search_page.dart` 的 widget 层

## 非目标

本轮不做：

1. 记忆折叠状态
2. 动画编排优化
3. 多级折叠
4. 按字段类型分栏

## 测试设计

至少覆盖：

1. 默认只显示命中结构 / 主导信号 / 提醒型结论
2. 默认不显示语义分层 / 字段分布 / 主导字段提示
3. 点击“展开更多观测”后显示隐藏内容
4. 点击“收起观测详情”后再次隐藏隐藏内容

## 完成后的转向建议

这一轮完成后，更适合转向：

1. 提升搜索结果本身的语义质量
2. 针对 detail page 做更强的命中上下文解释

不建议继续在顶部摘要层堆更多静态说明文案。
