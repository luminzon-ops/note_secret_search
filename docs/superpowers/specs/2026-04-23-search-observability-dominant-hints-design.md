# 搜索观测摘要主导提示设计

## 背景

当前 SearchPage 顶部已经具备：

1. 检索链路说明
2. 结果构成说明
3. 搜索观测摘要
4. 语义分层说明
5. 字段分布说明

这些信息已经足够支撑基础观测，但仍偏“原始统计”，缺少一层更直接的结论表达。

## 目标

在现有“搜索观测摘要”中增加两类简洁结论：

1. 主导信号提示
2. 主导字段提示

让用户或开发者能更快看出：

1. 当前结果主要由哪类命中信号主导
2. 当前语义命中主要集中在哪个字段

## 非目标

本次不做：

1. 不做图表或百分比
2. 不做展开详情
3. 不做多字段并列展示
4. 不做新的交互入口
5. 不把观测摘要升级成 dashboard

## 方案

采用：

- **混合型提示**

即在现有统计信息之后追加结论型文本，并保留括号中的计数。

## 放置位置

继续放在现有 `搜索观测摘要` 内，不新增新卡片。

顺序：

1. 命中结构
2. 语义分层
3. 字段分布（若有）
4. 主导信号提示
5. 主导字段提示（若有）

## 主导信号规则

从以下三类计数中取数量最多者：

1. 双命中
2. 关键词优先
3. 语义辅助

文案采用结论 + 计数，例如：

1. `当前结果主要由双命中主导（4 条）。`
2. `当前结果主要由关键词命中主导（3 条）。`
3. `当前结果主要由语义召回主导（2 条）。`

### 平局规则

固定优先级：

1. 双命中
2. 关键词优先
3. 语义辅助

这样可以优先输出更强信号，避免平局时提示不稳定。

## 主导字段规则

仅在存在语义字段分布时显示。

从语义字段计数中取数量最多者，文案例如：

1. `当前语义命中主要集中在标题字段（2 条）。`
2. `当前语义命中主要集中在标签字段（3 条）。`

### 平局规则

固定优先级：

1. 标题
2. 账号
3. 摘要
4. 网址
5. 附注
6. 正文
7. 标签

目的：

1. 优先高价值字段
2. 保证输出稳定

## 不显示情况

### 主导信号提示

始终显示，因为命中结构总是可计算。

### 主导字段提示

当没有语义字段参与时不显示。

## 结构建议

继续在：

- `lib/features/search/presentation/search_result_explanation.dart`

中扩展 `SearchObservabilitySummary`，新增：

1. `dominantSignalHint`
2. `dominantFieldHint`

SearchPage 只负责渲染，不负责拼接这些结论文案。

## 测试设计

至少覆盖：

1. keyword-only 场景主导信号提示正确
2. mixed 场景主导信号提示正确
3. 有语义字段时主导字段提示正确
4. 无语义字段时不显示主导字段提示
5. 平局时优先级符合预期

## 默认假设

1. 当前观测摘要仍以“轻量文本结论”为主，不追求图形化。
2. 主导提示用于快速判断，不承担完整分析职责。
3. 通过固定优先级处理平局，比展示多个并列主导更适合 MVP。

## 完成后的转向建议

这一轮收口后，更适合转向：

1. 搜索观测摘要中的更多质量结论（例如高质量语义占比趋势）
2. 更真实的语义质量提升

不建议继续无限扩充顶部摘要的纯文案层数。
