# 搜索观测摘要提醒型结论设计

## 背景

当前 SearchPage 的“搜索观测摘要”已经具备：

1. 命中结构
2. 语义分层
3. 字段分布
4. 主导信号提示
5. 主导字段提示

这些信息已经能帮助观察当前检索状态，但仍然偏“统计 + 结论”，还缺少一层更有行动意义的提醒。

## 目标

为现有观测摘要增加轻量的提醒型结论，帮助用户或开发者更快做出判断：

1. 当前语义参与是否偏弱
2. 当前语义命中是否主要来自辅助字段
3. 当前语义结果是否更多集中在高价值字段

## 非目标

本次不做：

1. 不做复杂规则引擎
2. 不做多个提醒同时堆叠
3. 不做颜色等级分类
4. 不做百分比图表
5. 不做 dashboard

## 方案

采用：

- **谨慎提示型提醒**

优先输出保守、风险提示式结论，而不是积极鼓励式提示。

## 放置位置

继续放在现有 `搜索观测摘要` 内，位于主导信号 / 主导字段提示之后。

顺序：

1. 命中结构
2. 语义分层
3. 字段分布（若有）
4. 主导信号提示
5. 主导字段提示（若有）
6. 提醒型结论（若有）

## 提醒规则

本轮只输出一条提醒，按固定优先级命中第一个规则后停止。

### 规则 1：关键词主导且语义参与弱

当满足：

1. 主导信号为 `关键词优先`
2. 且语义辅助或双命中较弱

输出：

- `当前结果主要由关键词命中主导，语义链路参与较弱。`

### 规则 2：语义参与主要来自辅助字段

当满足：

1. 存在语义字段参与
2. 主导字段属于辅助字段：
   - `网址`
   - `附注`
   - `正文`
   - `标签`

输出：

- `当前语义参与主要来自辅助字段，建议谨慎判断结果质量。`

### 规则 3：高价值字段集中

当满足：

1. 主导字段属于高价值字段：
   - `标题`
   - `账号`
   - `摘要`
2. 且高质量语义命中数量大于辅助语义命中数量

输出：

- `当前语义命中集中在高价值字段，可优先检查前排结果。`

## 优先级

提醒规则按以下顺序判定：

1. 关键词主导且语义参与弱
2. 语义参与主要来自辅助字段
3. 高价值字段集中

只显示第一条命中的提醒。

## 不显示情况

当以上规则都不命中时，不显示提醒型结论。

## 结构建议

继续在：

- `lib/features/search/presentation/search_result_explanation.dart`

中扩展 `SearchObservabilitySummary`，新增：

1. `reminderHint`

提醒逻辑仍放 helper 层，SearchPage 只渲染。

## 测试设计

至少覆盖：

1. keyword 主导弱语义场景输出关键词主导提醒
2. 辅助字段主导场景输出谨慎判断提醒
3. 高价值字段 + 高质量语义占优场景输出优先检查提醒
4. 无规则命中时不显示提醒
5. 多规则可能同时命中时按优先级只显示一条

## 默认假设

1. MVP 阶段提醒应优先服务快速判断，不追求完美精细的质量模型。
2. 谨慎提示比积极背书更安全。
3. 单条提醒比多条同时展示更适合当前顶部摘要密度。

## 完成后的转向建议

这一轮收口后，更适合转向：

1. 真实语义质量继续提升
2. 搜索观测摘要过长时的精简与分级展示

不建议无限向顶部摘要继续堆更多静态文案。
