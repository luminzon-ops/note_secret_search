# unified search 排序与语义质量提升设计

## 背景

当前 unified search 的融合排序逻辑主要基于：

1. 来源优先级（双命中 > 纯语义 > 纯关键词）
2. semanticScore
3. semantic 字段优先级
4. favorite
5. updatedAt

这个顺序已经能区分大多数结果，但仍存在一个问题：

- 高分但仅来自辅助语义字段的结果，可能排到低分但高质量字段双命中的结果前面。

## 目标

让 unified result 排序更稳定体现“结果质量”：

1. 双命中优先于弱语义召回
2. 高质量语义字段优先于辅助语义字段
3. semanticScore 仍保留，但不凌驾于基础质量层级之上

## 方案

在 `SearchFusionService` 中引入一个更明确的排序分层：

1. 命中来源层级
2. 语义质量层级
3. semanticScore
4. 字段优先级
5. favorite
6. updatedAt

## 排序层级

### 一级：命中来源层级

- 双命中 > 纯语义 > 纯关键词

### 二级：语义质量层级

对于包含语义信号的结果：

- 高质量字段（title / username / summary） > 辅助字段（url / secretNote / tags / noteBody） > 无字段

### 设计意图

这意味着：

- 双命中 + 高质量字段 应稳定排在 纯语义 + 辅助字段 之前
- 双命中 + 辅助字段 仍优先于纯关键词
- 纯语义内部再用语义质量和 score 排序

## 非目标

本轮不做：

1. 学习型排序模型
2. 多维打分公式公开展示
3. 改写 SearchPage 文案
4. 修改 detail page 行为

## 测试设计

至少覆盖：

1. 双命中 + 高质量字段 排在纯语义 + 辅助字段前
2. 双命中 + 辅助字段 排在纯关键词前
3. 纯语义内部高质量字段排在辅助字段前
4. 同层级下 score 更高者仍排前

## 完成后的转向建议

这一轮完成后，更适合转向：

1. 搜索结果去噪与阈值优化
2. 更真实的 semantic quality policy 收紧

不建议继续只靠解释文案补足排序问题。
