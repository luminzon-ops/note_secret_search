# 详情页搜索命中上下文解释增强设计

## 背景

当前 secret / note 详情页从 SearchPage 跳转进入时，顶部会显示：

1. 来自搜索
2. 命中方式
3. 查询词
4. 命中说明

同时页面内部已经支持根据 `searchContext` 自动滚动并高亮目标字段。

问题在于：用户虽然知道“命中了什么”，但还缺少一层更明确的行动导向：

- 我为什么会来到这个字段？
- 进入后应该先看哪一块？

## 目标

在不改变当前 handoff 结构的前提下，增加一条**优先查看**解释，让用户进入详情页后能马上理解该先看哪里。

## 方案

从 `searchContext` 解析目标字段，并在顶部 handoff 卡增加一条：

- `优先查看：...`

## 文案映射

### Secret detail

- `标题：...` → `优先查看：标题，这是当前最直接的命中位置。`
- `账号：...` → `优先查看：账号字段，这里最可能承载本次命中。`
- `网址：...` → `优先查看：网址字段，确认站点是否与查询相关。`
- `标签：...` → `优先查看：标签字段，确认分类或标签线索。`
- `备注：...` → `优先查看：备注字段，查看补充说明是否匹配查询意图。`

### Note detail

- `标题：...` → `优先查看：标题，这是当前最直接的命中位置。`
- `摘要：...` → `优先查看：摘要，先确认概要是否对应本次查询。`
- `标签：...` → `优先查看：标签字段，确认标签线索是否匹配。`
- `正文：...` → `优先查看：正文内容，这里最可能包含本次命中上下文。`

### 无法识别时

不显示 `优先查看` 行。

## 结构原则

1. 保持 handoff 卡结构不变，只增加一行说明
2. 解析逻辑放在 detail page 内部私有 helper 即可
3. 不引入新的 shared model，避免过度抽象

## 非目标

本轮不做：

1. 多段解释
2. 命中置信度分级
3. 不同来源的复杂模板系统
4. 改写 detail page 高亮机制

## 测试设计

至少覆盖：

1. Secret detail 标题命中时显示 `优先查看：标题...`
2. Secret detail 备注命中时显示备注导向说明
3. Note detail 摘要命中时显示摘要导向说明
4. Note detail 正文命中时显示正文导向说明
5. 未知 searchContext 时不显示 `优先查看`

## 完成后的转向建议

这一轮完成后，更适合转向：

1. detail page 中命中字段局部的更细粒度解释
2. 真正改进搜索排序与语义质量

不建议继续在 handoff 卡堆太多说明文案。
