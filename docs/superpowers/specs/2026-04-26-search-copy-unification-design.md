# Search Explanation Copy Unification 设计

## 背景

当前搜索链路里，同一类语义结果在不同位置使用了不同说法：

1. 结果列表标签使用“语义辅助”
2. 结果卡片解释使用“高质量语义命中 / 辅助语义线索”
3. 观测摘要使用 `semantic-only`、`高分保留线` 等偏技术化表达
4. 详情页承接说明使用“保留的语义命中”

这些文案单独看都能理解，但放在一起时，用户需要自己推断它们是否在描述同一件事。

## 目标

把列表页、观测摘要、详情页对语义结果的解释统一成同一套中文术语，让用户能更稳定地理解：

1. 哪些结果是关键词命中
2. 哪些结果是语义命中
3. 哪些语义命中更值得优先查看
4. 哪些语义结果只是补充线索，应该继续确认

## 方案

本轮采用“产品化中文术语优先”的统一方向：

1. 保留 `双命中`、`关键词优先`
2. 将原 `语义辅助` 统一收敛为 `语义命中`
3. 将原 `高质量语义命中` 统一收敛为 `重点语义命中`
4. 将原 `辅助语义命中 / 辅助语义线索` 统一收敛为 `补充语义线索`
5. 将 observability 中的 `semantic-only` 改成中文表述，避免中英混用

## 文案映射

### 列表结果标签

- `dual` → `双命中`
- `keywordPrimary` → `关键词优先`
- `semanticAssist` → `语义命中`

### 结果卡片解释

- 双命中 + 重点语义字段：`这条结果同时命中关键词与重点语义字段，可优先查看。`
- 双命中 + 补充语义字段：`这条结果同时命中关键词，语义部分主要提供补充线索，建议结合预览确认。`
- 关键词优先：`这条结果主要由关键词命中进入结果。`
- 纯语义 + 重点语义字段：`这条结果主要由重点语义命中支持，适合优先检查。`
- 纯语义 + 补充语义字段：`这条结果主要由补充语义线索召回，建议继续确认。`

### 观测摘要

- `命中结构` 中把 `语义辅助` 改为 `语义命中`
- `语义分层` 中把 `高质量` 改为 `重点`，把 `辅助` 改为 `补充线索`
- 语义过滤统计改为：
  - `语义过滤：语义直达候选 X 条，保留 Y 条，过滤 Z 条。`
- 过滤原因改为：
  - `过滤原因：被过滤结果多数只提供补充语义线索，且未达到高分保留条件。`

### 详情页承接说明

- `searchSource == 'semantic'` 时：
  - `承接说明：该结果以语义命中进入详情页，建议结合命中字段与正文继续确认。`

## 结构原则

1. 优先复用现有 helper，不重做搜索逻辑
2. 统一重点放在“术语”与“结论句式”，不扩展状态对象
3. 列表页与详情页的命中方式标签必须共用同一套叫法

## 非目标

本轮不做：

1. 调整底层搜索排序或过滤逻辑
2. 新增字段级 explainability 维度
3. 修改搜索结果路由参数结构
4. 扩展更多颜色/等级 UI

## 测试设计

至少覆盖：

1. `buildSearchResultCardExplanation` 输出新的统一化文案
2. `buildSearchObservabilitySummary` 输出新的统一化命中结构/语义分层/过滤原因文案
3. `SearchPage` 渲染新的 `语义命中` 标签和统一后的解释文案
4. `SecretDetailPage` / `NoteDetailPage` 显示新的详情页承接说明

## 完成后的转向建议

这一轮完成后，更适合转向：

1. 将“排序依据”与“语义命中”区块也做术语统一
2. 继续收敛 detail page 的字段提示文案，使其与结果卡片解释更一致
